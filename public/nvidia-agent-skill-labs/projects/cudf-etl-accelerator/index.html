Project Purpose

把 pandas 型資料清理流程重新設計成 GPU DataFrame 加速工作流，展示資料工程與 GPU computing 的連結。

Target Scenario

8000 萬筆訂單 pandas ETL 需 40 分鐘，改用 cuDF 後 A100 上只需 90 秒。

Proposed Architecture

  1. 定義 input data、model、asset、document 或 video stream。
  2. 建立 baseline workflow 與假設條件。
  3. 導入對應 NVIDIA skill 作為 acceleration、training、deployment 或 simulation layer。
  4. 用 benchmark、demo walkthrough 或 checklist 驗證結果。
  5. 整理成 GitHub README、demo page 與 LinkedIn 摘要。

Deliverables

  • CPU vs GPU ETL benchmark notebook
  • 資料清理 pipeline
  • 效能比較圖表
  • LinkedIn 專案摘要

Milestones

  1. 建立 pandas baseline
  2. 導入 cuDF API 對照
  3. 加入批次資料與 Parquet 輸出
  4. 整理效能報告與 README
