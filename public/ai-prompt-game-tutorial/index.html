01

為什麼我會用遊戲來教 AI Prompt

遊戲很適合拿來訓練 AI Prompt，因為遊戲需要明確的規則、角色、限制、狀態、回饋與勝利條件。 這些元素剛好也是跟 AI 溝通時最容易缺少的部分。當我把一個模糊想法變成遊戲，學員會很快感受到： prompt 不是一句命令，而是一份可以逐步測試的設計文件。

Prompt Thinking

把想法變成可執行規格

先描述玩家、目標、限制與輸出格式，讓 AI 有清楚的設計邊界。

Game Loop

把規則變成互動循環

把「玩家做什麼、系統如何回應、如何判斷成功」拆成可以測試的流程。

Portfolio Ready

把練習變成作品集

最後整理成 GitHub、網頁 demo、LinkedIn 文章與課程截圖，讓成果可以被看見。

我的教學重點： 我會讓學員先完成一個很小、很清楚、可以跑起來的小遊戲，再逐步加入圖片、音效、資料紀錄或 AI 角色。
02

課程流程

這份流程可以做成 30 分鐘快速體驗，也可以展開成完整 workshop。每個階段都會產出一小段可保存的 prompt。

00:00 - 05:00

從遊戲想法開始

選一個簡單概念，例如記憶卡牌、接物遊戲、問答闖關、節奏反應或迷宮任務。

05:00 - 12:00

寫出第一版遊戲企劃 Prompt

要求 AI 回傳玩法、規則、勝利條件、失敗條件、玩家回饋與畫面元素。

12:00 - 22:00

把企劃轉成可玩的網頁原型

請 AI 依照規格生成 HTML、CSS、JavaScript，並要求手機與桌面都能操作。

22:00 - 32:00

測試、除錯、調整手感

用「看到的現象」回報給 AI，例如按鈕無反應、分數沒有增加、手機版畫面太擠。

32:00 - 45:00

包裝成作品集

整理 GitHub README、Demo 連結、截圖、遊戲設計說明與 LinkedIn 發文摘要。

03

我的 Prompt 架構

我習慣把 prompt 拆成五個區塊：角色、目標、輸入資料、限制、輸出格式。這樣 AI 比較不會只回一段空泛建議， 而是能產出可用的企劃、程式碼或檢查清單。

Role

你現在扮演誰

例如遊戲設計師、前端工程師、教學設計師、QA 測試員。

Goal

我要你完成什麼

例如設計 3 分鐘可玩的遊戲、生成原型、檢查規則漏洞。

Output

你要用什麼格式回覆

例如表格、條列、單一 HTML 檔、README、LinkedIn 貼文。

04

Prompt Pack

下面是可以直接拿去上課或示範的 prompt 範本。每一段都保留可替換欄位，方便改成不同主題。

遊戲概念生成 
你是一位遊戲設計師與 AI 教學講師。
請幫我設計一個適合初學者在 30 分鐘內完成的網頁小遊戲。

主題：[填入主題，例如語言記憶、AI 工具、半導體知識]
玩家：[填入對象，例如高中生、求職者、產品經理]
限制：
- 規則要簡單
- 可以用 HTML/CSS/JavaScript 做出第一版
- 手機與桌面都能玩

請用表格輸出：
1. 遊戲名稱
2. 核心玩法
3. 勝利條件
4. 失敗條件
5. 畫面元素
6. 可以延伸的 AI 功能
規則拆解 
請把以下遊戲想法拆成可開發規格。

遊戲想法：
[貼上你的遊戲概念]

請輸出：
- 玩家可以做的動作
- 系統需要記錄的狀態
- 每回合發生什麼事
- 分數如何計算
- 哪些情況會讓玩家困惑
- 第一版最小可行功能

請避免加入太複雜的功能，先讓遊戲可以完成一輪。
生成遊戲原型 
你是一位前端工程師。
請依照以下規格，產生一個單檔 HTML 小遊戲原型。

規格：
[貼上規則拆解結果]

技術限制：
- 只使用 HTML、CSS、JavaScript
- 不使用外部套件
- 版面需要支援手機與桌面
- 按鈕、分數、重新開始都要可以操作

請直接輸出完整程式碼，並在程式碼後列出三個可以改善手感的方向。
測試與除錯 
我正在測試這個小遊戲，遇到以下問題。

看到的現象：
[例如按下開始沒有反應 / 分數沒有增加 / 手機版文字超出畫面]

我預期的行為：
[描述你希望它怎麼運作]

請幫我：
1. 推測可能原因
2. 指出需要修改的程式區塊
3. 給我修正後的完整程式碼
4. 補上測試清單，確認問題真的修好
05

示範遊戲原型：Prompt Quest

這是一個示範概念：玩家要收集三個 prompt 節點，分別代表角色、限制與輸出格式，最後抵達完成線。 這種視覺化遊戲很適合教新手理解「好的 prompt 是一套設計流程」。

玩家收集 Role、Constraint、Output 三個節點後，才能完成任務。這可以延伸成真正的小遊戲，也可以當作課堂中的互動示意。

如何把它做成作品

第一版只需要移動、碰撞、分數與勝利條件。第二版再加入音效、提示卡、排行榜或 AI 生成關卡。

  • 用 prompt 生成遊戲規格與 UI 文案。
  • 請 AI 產生單檔 HTML 原型，先確定能玩。
  • 把測試過程、錯誤修正與迭代截圖寫進 README。
  • 把完成品整理成 LinkedIn 貼文與 Featured 連結。
06

練習任務

課後 3 件事

這些任務會把練習從「用 AI 做出東西」推進到「整理成可以公開展示的作品」。

  • 完成一個 1 分鐘內能玩完的小遊戲，並記錄 AI 幫忙完成了哪些部分。
  • 建立 GitHub README：放遊戲目標、操作方式、Prompt 摘要、Demo 截圖與後續計畫。
  • 寫一段 LinkedIn 摘要：說明你如何用 prompt 把想法變成可測試的遊戲原型。
07

思考題

Question 1

AI 幫你做太多時，你還能展示什麼能力？

可以展示問題定義、規格拆解、測試修正、取捨判斷與作品包裝。

Question 2

好的遊戲 prompt 和好的產品需求有什麼相似？

兩者都需要清楚的使用者、限制、成功標準、回饋流程與迭代方式。

Question 3

要如何避免 AI 做出看起來很炫但不好玩的遊戲？

先定義最小互動循環，測試手感與理解成本，再增加視覺效果。
